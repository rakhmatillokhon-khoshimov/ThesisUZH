Large language model audits often use provider APIs because API access is reproducible and easy to automate. Ordinary users interact with consumer applications that may add system instructions, tools, retrieval, session handling, location context, and visible interface features. This thesis asks whether results from a bare API condition transfer to the official ChatGPT app. The design is a black box comparison rather than a causal test of access channel. It compares the OpenAI Responses API using GPT 4o without a custom system prompt with the official ChatGPT app, whose hidden product stack cannot be separated into system prompt, tools, retrieval, routing, and session state.

Using a frozen 60 prompt suite across six categories, responses are coded for refusal status, safety framing, instruction following, unsupported inference, factuality, verbosity, and visible UI surface signatures. Manual review identifies 17 substantive observable divergences. Ten have response core anchors, including six refusal boundary cases, three factual support cases, and one format compliance case. Seven are product surface or context cases such as visible sources or localized context. The primary refusal asymmetry is directional but not conventionally significant with McNemar exact p = 0.062, so the statistical result is treated as underpowered. Two additional API repetitions show perfect stability for the OpenAI API refusal label with Fleiss kappa = 1.0, and API repetitions are lexically closer to each other than to the app. A targeted app repetition check adds two fresh ChatGPT app runs for each final case. Fourteen of 17 original anchors reproduce in both repetitions, one reproduces once, and two do not reproduce. All six refusal boundary cases and all three factual support cases reproduce twice.

An independent blind second coder pass on 54 coding units shows strong agreement for refusal status, format compliance, unsupported inference, and acceptable agreement for factuality. UI surface agreement is moderate, while safety framing is weak and is therefore used only as descriptive context. The contribution is a compact reproducible protocol and a conservative case based audit of observable deployment surface differences, not a population estimate or causal decomposition of hidden product mechanisms.
